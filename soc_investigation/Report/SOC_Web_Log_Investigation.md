# SOC Investigation Report: Suspicious Web Traffic Analysis Using Splunk

## Executive Summary

A security investigation was conducted on web server HTTP logs using Splunk to identify potential indicators of malicious activity. The analysis revealed multiple suspicious events, including SQL Injection reconnaissance attempts, Directory Traversal activity, Web Shell access attempts, unusual HTTP methods, suspicious user agents, and abnormal file transfer activity.

Several requests were generated by tools commonly associated with offensive security testing and automated attacks. The findings indicate that the target web application was subjected to reconnaissance and potential exploitation attempts.

Based on the evidence collected, the overall risk level of the observed activity is assessed as **High**.

---

# Investigation Scope

### Objective

Analyze HTTP logs to identify:

* Suspicious web requests
* Attack indicators
* Malicious user agents
* Unauthorized resource access attempts
* Potential exploitation activity

### Data Source

* HTTP Access Logs
* JSON Format
* Analyzed using Splunk Enterprise

### Investigation Platform

* Splunk Enterprise
* SPL (Search Processing Language)

---

# Key Findings

## Finding 1: SQL Injection Reconnaissance Activity

### Severity

High

### Description

Analysis identified requests generated by the SQLMap tool.

Detected User Agent:

```text
sqlmap/1.5.1
```

SQLMap is an automated SQL Injection testing framework commonly used to discover and exploit database vulnerabilities.

### Evidence

```spl
index=web_practicing user_agent="*sqlmap*"
| table _time id.orig_h method uri user_agent status_code
```

### Impact

Successful SQL Injection attacks may result in:

* Database compromise
* Data theft
* Credential exposure
* Unauthorized access

### MITRE ATT&CK

T1190 - Exploit Public-Facing Application

---

## Finding 2: Directory Traversal Attempt

### Severity

Critical

### Description

Requests targeting sensitive operating system files were identified.

Detected Resource:

```text
/etc/passwd
```

The file is commonly targeted during Directory Traversal and Local File Inclusion attacks.

### Evidence

```spl
index=web_practicing uri="*/etc/passwd*"
| table _time id.orig_h method uri user_agent status_code
```

### Impact

Potential consequences include:

* User enumeration
* Information disclosure
* System reconnaissance

### MITRE ATT&CK

T1006 - Path Traversal

---

## Finding 3: Web Shell Access Attempt

### Severity

Critical

### Description

Requests targeting a potential web shell resource were observed.

Detected Resource:

```text
/ shell.php
```

Web shells are commonly used by attackers to maintain persistence and execute remote commands.

### Evidence

```spl
index=web_logs uri="*/shell.php*"
| table _time id.orig_h method uri user_agent status_code
```

### Impact

Potential consequences include:

* Remote command execution
* Persistence
* Data theft
* Full server compromise

### MITRE ATT&CK

T1505.003 - Web Shell

---

## Finding 4: Suspicious User Agents

### Severity

High

### Description

Multiple non-browser user agents were identified.

Observed Tools:

* sqlmap
* curl
* python-requests
* botnet-checker

These tools are commonly associated with automated reconnaissance and attack activity.

### Evidence

```spl
index=web_logs
| search user_agent="*sqlmap*" OR user_agent="*curl*" OR user_agent="*python-requests*" OR user_agent="*botnet*"
| table _time id.orig_h user_agent uri status_code
```

### Impact

Indicates active probing and automated interaction with the application.

---

## Finding 5: Unexpected HTTP Methods

### Severity

Medium

### Description

Several uncommon HTTP methods were detected.

Observed Methods:

* PUT
* DELETE
* OPTIONS
* CONNECT

These methods may indicate reconnaissance, file upload attempts, resource manipulation, or tunneling activity.

### Evidence

```spl
index=web_logs
event_type="Unexpected Method"
| stats count by method
```

### Impact

Potential attack surface discovery and exploitation attempts.

---

## Finding 6: Server and Client Errors

### Severity

Medium

### Description

Multiple HTTP error responses were observed.

Detected Status Codes:

* 400 Bad Request
* 404 Not Found
* 500 Internal Server Error
* 503 Service Unavailable

The pattern is consistent with automated scanning and vulnerability testing.

### Evidence

```spl
index=web_logs
| stats count by status_code
```

### Impact

Potential indication of malformed requests, enumeration activity, or attack-related application instability.

---

## Finding 7: Large Transfer Events

### Severity

Medium

### Description

Several unusually large HTTP responses were identified.

Observed Transfers:

* Greater than 1 MB
* Up to approximately 1.9 MB

### Evidence

```spl
index=web_logs
resp_body_len > 1000000
| table _time id.orig_h uri resp_body_len status_code
```

### Impact

May indicate:

* Large file downloads
* Data exposure
* Potential data exfiltration

| Unexpected Methods       | Medium   |

### Overall Risk Rating

---

# Recommendations

1. Investigate all source IP addresses associated with suspicious activity.
2. Block identified malicious IPs where appropriate.
3. Review web server logs for successful exploitation attempts.
4. Implement Web Application Firewall (WAF) protections.
5. Disable unnecessary HTTP methods such as PUT and DELETE.
6. Apply input validation to prevent SQL Injection and Directory Traversal attacks.
7. Review server file systems for unauthorized PHP files.
8. Enable continuous monitoring and alerting within Splunk.
9. Conduct vulnerability assessments against public-facing applications.
10. Implement least-privilege access controls and secure configuration practices.

---

# Conclusion

The investigation identified multiple indicators of malicious reconnaissance and potential exploitation attempts against the target web application. Evidence of SQLMap activity, Directory Traversal attempts, Web Shell access requests, suspicious user agents, and abnormal HTTP methods suggests that external entities actively probed the environment for vulnerabilities.

While the available logs do not confirm successful compromise, the observed activity represents a significant security concern and warrants further monitoring, investigation, and remediation efforts.
