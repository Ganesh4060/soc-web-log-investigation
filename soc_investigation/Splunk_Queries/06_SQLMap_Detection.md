# 06 - SQLMap Detection and SQL Injection Reconnaissance

## Objective

The objective of this phase of the investigation is to identify evidence of SQL Injection reconnaissance activity through the detection of SQLMap user agents within the web traffic logs.

SQL Injection remains one of the most common and dangerous web application attacks. Attackers often use automated tools such as SQLMap to discover vulnerable parameters, enumerate databases, extract sensitive information, and potentially gain unauthorized access to backend systems.

The detection of SQLMap activity is a strong indicator of malicious reconnaissance and vulnerability assessment efforts.

---

## Splunk Query

```spl
index=web_practicing
user_agent="*sqlmap*"
| table _time id.orig_h method uri user_agent status_code
```

---

## Alternative Detection Query

```spl
index=web_practicing
event_type="Suspicious Agent"
| search user_agent="*sqlmap*"
| stats count by id.orig_h uri user_agent
```

This query identifies source hosts using SQLMap and the resources being targeted.

---

## Background

SQLMap is an open-source penetration testing tool used to automate SQL Injection detection and exploitation.

The tool is capable of:

* Database fingerprinting
* Database enumeration
* Table discovery
* User enumeration
* Password hash extraction
* Data dumping
* Remote command execution (in certain environments)

Because SQLMap automates the attack process, it is frequently observed during real-world attacks against web applications.

---

## Investigation Methodology

The investigation focused on identifying requests generated by SQLMap and analyzing associated activity.

The following indicators were examined:

* SQLMap user agents
* Targeted URIs
* Request methods
* HTTP response codes
* Source IP addresses
* Correlation with server errors

---

## Observed Activity

During analysis, requests were identified containing the following user agent:

```text
sqlmap/1.5.1
```

This user agent is directly associated with the SQLMap framework and strongly indicates intentional SQL Injection testing activity.

Unlike normal browsers, SQLMap is specifically designed to discover and exploit database vulnerabilities.

---

## Security Analysis

The presence of SQLMap traffic indicates that an individual or automated system was actively assessing the web application for SQL Injection vulnerabilities.

Common attacker objectives include:

* Identifying injectable parameters
* Enumerating backend databases
* Accessing sensitive information
* Obtaining administrative credentials
* Escalating privileges
* Achieving remote code execution

This behavior is not consistent with normal user activity and should be treated as malicious reconnaissance.

---

## Potential Attack Scenario

A typical SQL Injection attack may follow the sequence below:

### Stage 1 – Reconnaissance

The attacker identifies web pages that accept user input.

Examples:

```text
/login
/search
/product?id=1
/account
```

---

### Stage 2 – Automated Testing

SQLMap sends specially crafted requests to determine whether SQL Injection vulnerabilities exist.

Example payloads:

```sql
' OR 1=1 --
```

```sql
UNION SELECT NULL,NULL
```

```sql
AND 1=1
```

---

### Stage 3 – Database Enumeration

If a vulnerability is discovered, the attacker may attempt to retrieve:

* Database names
* Tables
* Columns
* User accounts

---

### Stage 4 – Data Extraction

Sensitive information may be extracted, including:

* Credentials
* Personal data
* Financial records
* Application secrets

## Threat Hunting Recommendations

The following investigative actions are recommended:

1. Identify all source IPs associated with SQLMap activity.
2. Review targeted application endpoints.
3. Investigate server-side errors associated with SQLMap requests.
4. Search for SQL Injection payloads within URI parameters.
5. Review database logs for suspicious queries.
6. Validate application input sanitization controls.
7. Monitor for continued SQLMap activity.

---

## Key Findings

The investigation identified clear evidence of SQLMap activity targeting the web application.

Key observations include:

* The SQLMap user agent was detected in HTTP requests.
* Activity is consistent with SQL Injection reconnaissance.
* Attackers attempted to identify vulnerable application parameters.
* The observed behavior aligns with known web application attack techniques.
* The activity represents a significant security risk to backend systems and stored data.

---

## Conclusion

Analysis of the HTTP logs revealed requests generated by SQLMap, a tool specifically designed to automate SQL Injection testing and exploitation. The presence of SQLMap strongly suggests that an attacker was actively assessing the application for database-related vulnerabilities.

Given the potential impact of successful SQL Injection attacks, including unauthorized database access and sensitive data exposure, this activity should be considered a critical security finding and investigated further.
