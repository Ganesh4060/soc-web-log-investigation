# 04 - Directory Traversal Detection and Analysis

## Objective

The objective of this phase of the investigation is to identify and analyze potential Directory Traversal attacks observed within the web server logs. Directory Traversal is a common web application attack technique that allows an attacker to access files and directories outside the intended web application directory structure.

Successful Directory Traversal attacks can expose sensitive operating system files, configuration files, credentials, source code, and other critical information that may assist attackers in further compromising a target environment.

---

## Splunk Query

```spl
index=web_practicing
uri="*/etc/passwd*"
| table _time id.orig_h method uri user_agent status_code
```

---

## Alternative Detection Query

```spl
index=web_practicing
| search uri="*/etc/passwd*" OR uri="*../*" OR uri="*..%2f*" OR uri="*..\\*"
| table _time id.orig_h method uri user_agent status_code
```

This query helps identify common Directory Traversal patterns including:

* ../
* ..\
* URL-encoded traversal sequences
* Access attempts targeting sensitive files

---

## Background

Directory Traversal, also known as Path Traversal, occurs when an attacker manipulates file path input parameters to access files located outside the application's intended directory.

Common attacker targets include:

```text
/etc/passwd
/etc/shadow
/etc/hosts
boot.ini
web.config
application.properties
config.php
```

These files may contain valuable information that can assist attackers in privilege escalation, credential harvesting, or further exploitation.

---

## Investigation Methodology

The investigation focused on identifying requests targeting sensitive system files and suspicious path traversal patterns.

The following indicators were reviewed:

* Requests containing "../"
* Requests targeting operating system files
* Encoded traversal sequences
* Requests generated by suspicious user agents
* Associated HTTP response codes

---

## Observed Activity

During analysis, requests were identified attempting to access:

```text
/etc/passwd
```

The file `/etc/passwd` is a critical Linux system file that contains user account information.

Legitimate web applications rarely require direct access to this file through HTTP requests.

As a result, requests targeting this resource are considered highly suspicious.

---

## Security Analysis

The request targeting `/etc/passwd` strongly indicates an attempt to determine whether the web application is vulnerable to Directory Traversal or Local File Inclusion (LFI) attacks.

Attackers commonly use this technique because:

* The file exists on most Linux systems.
* The file is readable by default.
* Successful retrieval confirms access to the underlying operating system.
* Retrieved usernames can support future attacks.

This activity is consistent with reconnaissance and vulnerability assessment behavior.

---

## Potential Attack Scenario

An attacker may attempt to exploit insufficient input validation within the application by manipulating file path parameters.

Example:

```text
https://target.com/download?file=../../../../etc/passwd
```

If successful, the server may return sensitive system information to the attacker.

Potential consequences include:

* User enumeration
* System fingerprinting
* Information disclosure
* Credential harvesting
* Further exploitation opportunities

---

## Correlation with Other Findings

The Directory Traversal activity was observed alongside other suspicious indicators identified during the investigation, including:

* SQLMap user agent activity
* Curl-based requests
* Python automation tools
* Suspicious URI access attempts
* Unexpected HTTP methods

The combination of these indicators suggests coordinated reconnaissance against the target environment rather than normal user behavior.

## Threat Hunting Recommendations

The following actions are recommended:

1. Identify all source IP addresses associated with traversal attempts.
2. Review requests preceding and following the detected activity.
3. Investigate whether any requests returned HTTP 200 responses.
4. Search for additional traversal payloads.
5. Verify that file access controls are properly configured.
6. Implement input validation and path normalization controls.
7. Monitor future requests targeting sensitive operating system files.

---

## Key Findings

The investigation identified clear evidence of Directory Traversal reconnaissance activity.

Key observations include:

* Requests targeting the sensitive Linux file `/etc/passwd`.
* Behavior consistent with web application vulnerability scanning.
* Potential attempts to access files outside the application's intended directory structure.
* Activity aligned with known attacker reconnaissance techniques.
* Elevated risk due to the nature of the targeted resource.

---

## Conclusion

Analysis of HTTP request data revealed attempts to access sensitive operating system files through Directory Traversal techniques. The request targeting `/etc/passwd` is a strong indicator of malicious reconnaissance and suggests that an attacker was evaluating the application for file disclosure vulnerabilities.

Although log analysis alone cannot confirm successful exploitation, the observed activity represents a significant security concern and should be treated as a high-priority indicator of compromise. Additional investigation and continuous monitoring are recommended to determine whether further exploitation attempts occurred.
